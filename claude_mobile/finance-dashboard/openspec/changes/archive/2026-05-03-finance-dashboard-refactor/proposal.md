## Why

Форма ритуала финансового дашборда имеет дублирующиеся поля (Выплата + Тип события), ручной ввод НДФЛ рядом с расчётным значением, нет ограничения "вычеты только на 5-е число", некорректное округление распределения и отсутствие видимости того, из каких категорий фактически покрываются обязательные траты.

## What Changes

- Убрать дублирование: кнопки "5-е/20-е" (payDay) удаляются, остаётся только dropdown "Тип события"
- Топ-блок ввода: Начислено + Бейдж + ДМС Тани + Страховка Тани + Перерасход — все в одном месте
- Раздел "Вычеты" заменяется одной read-only строкой НДФЛ (badge × 13%)
- Вычеты показываются автоматически только при типе `5th_payday`; для других типов — скрыты за кнопкой "+ Вычеты"
- Округление: `Math.floor` для всех категорий кроме "На себя"; "На себя" = остаток
- **BREAKING**: `pct` категории "На себя" больше не редактируется в настройках (становится "остатком")
- Маппинг вычетов на категории: новый state `deductionMapping` с UI в Настройках
- Таблица распределения получает колонки "Потрачено" и "Доступно" на основе маппинга

## Capabilities

### New Capabilities

- `deduction-mapping`: Настраиваемый маппинг вычетов (НДФЛ, ДМС, страховка, перерасход) на категории бюджета; отображение "потрачено/доступно" в таблице распределения
- `event-driven-deductions`: Автоматическое управление видимостью вычетов в зависимости от типа события (показывать для 5-го, скрывать для остальных)

## Impact

- `claude_mobile/finance-dashboard/index.html` — единственный файл; затрагиваются state-объявления, расчётные useMemo, UI ритуала (ЗП таб), UI настроек
