## Context

Нет серверной части. Приложение — один HTML-файл, работает из файловой системы или любого статического хостинга. Внешний API: picsum.photos — публичный, без ключей.

## Goals / Non-Goals

**Goals:**
- Один файл `index.html`, открывается напрямую в браузере
- Случайное изображение по нажатию кнопки
- Скачивание текущего изображения
- Нет зависимостей сборки

**Non-Goals:**
- Серверная часть
- Хранение истории изображений
- Фильтры, категории, теги
- Поддержка офлайн-режима

## Decisions

**1. API: picsum.photos вместо lorempixel / placekitten**
picsum.photos стабилен, поддерживает seed для воспроизводимости, отдаёт реальные фото.

**2. `<img src>` вместо fetch + blob**
Прямая ссылка в `src` работает без CORS-проблем. Seed меняется на `Date.now()` для гарантии нового изображения.

**3. Скачивание через `fetch` + `createObjectURL`**
`<a download href=url>` не работает для cross-origin. Fetch + blob + `a.click()` — корректное решение.

**4. Чистый CSS, без фреймворков**
Минимум кода, нет webpack/vite, файл открывается двойным кликом.

## Risks / Trade-offs

- [picsum.photos недоступен] → изображение не загрузится. Mitigation: показать сообщение об ошибке.
- [CORS при скачивании] → fetch может быть заблокирован. Mitigation: обработать ошибку, показать ссылку вместо авто-скачивания.
- [Кэш браузера] → одна и та же картинка. Mitigation: добавить timestamp как query-параметр.
