# Plan: Prompt Engineer — кликабельный прототип (скорректированный)

## Context

Есть v1 (prompt-engineer.jsx) — старая архитектура. Нужен полный переписка под новый флоу v3:
- Ветка A (создать): новый/продолжить → тип → наработки (опц.) → задача → 5 вопросов по одному
- Ветка B (улучшить): вставить промпт → разобрать по XML (B2a) ИЛИ преобразовать (B2b)
- Дизайн: новая токен-система (янтарный тёмный, #f59e0b акцент)

Цель: index.html — самодостаточный файл, открывается в браузере без билд-системы.

---

## Синтез проекта

Что строим: инструмент для написания и улучшения AI-промптов под **разные модели** (не только Claude). Ключевая ценность — снизить барьер входа: задал задачу через wizard → получил структурированный промпт; или вставил черновик → AI распарсил/улучшил.

Технические вызовы:
1. API — сейчас заглушка-мок, реальная интеграция придёт из другого проекта позже
2. Мультимодельность — архитектура состояния должна поддерживать выбор модели (Claude, GPT, Gemini и др.)
3. B2a DnD — HTML5 DnD десктоп-only; осторожная работа с dragOver/drop/dragStart
4. Парсинг XML из AI — JSON может прийти с markdown-обёрткой (```json...```). Нужен defensive parse
5. A3 пропуск — mode === 'new' → skip A3

---

## Решения

- **API**: заглушка-мок, возвращает правдоподобные фиктивные ответы. Реальный API подключается позже из `/projects/vibe/groovy_agent`
- **Модели**: UI предусматривает выбор модели — минимум селект с вариантами (Claude Sonnet, GPT-4o, Gemini 1.5 Pro и др.). Архитектура состояния хранит `selectedModel`
- **"Уточнить"**: API-рефайнмент — открывает поле "Что изменить?" → вызов к выбранной модели
- **Фон**: чистый, без сетки и glow. Фон = `var(--bg)` и ничего больше.
- **DnD в B2a**: реализовать — это MVP-требование
- **Ошибки**: try/catch + inline-сообщение без краша

## Пошаговый план реализации

### Sprint 1 — Скелет + дизайн-система (без API)

1. `index.html` с React 18 CDN + Babel standalone + Inter/Instrument Serif из Google Fonts
2. CSS-переменные для токенов темы (dark default)
3. Фон: просто `var(--bg)`, никаких сеток и glow
4. Header: логотип "Prompt Engineer" (Instrument Serif italic) + селект модели + кнопка сброса
5. Селект модели — выпадашка из MODEL_LIST, хранится в состоянии
6. ModeScreen — две большие карточки (Создать / Улучшить)
7. Навигация: `setStep()` + переходы по всем экранам-заглушкам

### Sprint 2 — Ветка A (все экраны)

8. A1 — Новое / Продолжить (2 карточки)
9. A2 — Контент / Код / Отчёт (3 карточки)
10. A3 — Наработки textarea + автопропуск если `mode === 'new'`
11. A4 — Задача textarea, счётчик символов, кнопка активна при ≥20 символов
12. A5 — Прогресс-бар 5 точек + текущий вопрос + превью следующего (серый) + 3 кнопки:
    - «← Назад» / «⚡ Генерировать сейчас» / «Далее →»
    - 5-й вопрос зависит от `a.type`
    - На 5-м вопросе «Далее →» → «Сгенерировать →»

### Sprint 3 — Ветка B + API-заглушка

13. B1 — textarea + 2 кнопки (активны при непустом поле): «🔍 Разобрать по тегам» / «✏️ Преобразовать»
14. B2b — 4 карточки трансформации (multi-select, без "Сменить язык") + своё поле + «Преобразовать →»
15. **Мок-функция** `callModel(system, user, model)` — имитирует задержку 1.5–2с, возвращает фиктивный правдоподобный промпт
16. Промпт генерации (ветка A) — строится из `a.*` → `callModel`
17. Промпт преобразования (B2b) — карточки + своё поле → `callModel`
18. LoadingScreen — спиннер + сообщение
19. ResultScreen:
    - `<pre>` с текстом + счётчик символов
    - «📋 Копировать» → «✓ Скопировано» 2 сек
    - «🌍 → EN» — мок перевода на английский
    - «↺ Уточнить» → textarea "Что изменить?" → `callModel` (рефайнмент)
    - «🔄 Начать заново» → полный сброс

### Sprint 4 — B2a XML редактор

20. Мок парсинга: промпт → `callModel` → defensive parse JSON (strip ```json```)
21. Рендер блоков: тег, textarea, кнопки [↕][✏️][🗑][📋]
22. Редактирование контента в textarea блока
23. Переименование тега — inline input по [✏️]
24. Удаление блока — [🗑]
25. Индикатор отсутствующих тегов из 6 стандартных (`role`, `context`, `task`, `format`, `constraints`, `examples`)
26. «📋 Скопировать весь XML»
27. HTML5 DnD — `draggable`, `onDragStart` / `onDragOver` / `onDrop`

### Sprint 5 — Полировка

28. Анимации: fadeUp для карточек (stagger +0.05s)
29. Мобильная адаптация (кроме DnD)
30. Ошибки: try/catch → inline error, кнопка «Попробовать ещё раз»
31. Горячие клавиши: Cmd+Enter = генерировать, Esc = назад

## Верификация

1. Открыть `index.html` в браузере без сервера — работает
2. A: Mode → A1 (Новое) → A2 (Код) → A4 → A5 → ⚡ → Result → Копировать
3. A: Mode → A1 (Продолжить) → A2 → A3 → A4 → A5 → Сгенерировать → Уточнить
4. B: Mode → B1 → Разобрать → B2a → редактировать → копировать XML
5. B: Mode → B1 → Преобразовать → B2b (2 карточки + своё) → Result
6. B2a: невалидный JSON → не крашится, показывает ошибку
7. Мобилка: layout работает, DnD не показывается
