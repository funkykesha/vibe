# Prompt Engineer — логика флоу v3

---

## Экран 0 — Выбор режима

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   🔬  Создать промпт          🔄  Улучшить промпт       │
│   Исследование или задача     У меня уже есть текст     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**→ Две независимые ветки**

# ВЕТКА A — Создать промпт

## A1 — Что делаем?

```
[ 🆕 Начинаю с нуля ]     [ 🔁 Продолжить / доделать ]
```

## A2 — Тип результата

```
[ ✍️ Контент ]    [ ⌨️ Код ]    [ 📄 Отчёт / документ ]
```

*(Контент = тексты, посты, статьи, письма; Отчёт = документ, анализ, ВКР)*

## A3 — Наработки

> ⚡ **«Новое» (A1)** → этот экран пропускается автоматически, переход сразу к A4
> 📂 **«Продолжить» (A1)** → экран показывается

```
┌─────────────────────────────────────────────────┐
│ Что уже есть?                                   │
│ Вставьте текст, опишите наработки, дайте ссылку │
│                                                 │
│ [textarea — свободный ввод]                     │
│                                                 │
└─────────────────────────────────────────────────┘
              [ Далее → ]   ← работает если пусто
```

## A4 — Задача

*Единственное обязательное поле (минимум ~20 символов)*

```
┌─────────────────────────────────────────────────┐
│ Опишите задачу своими словами                   │
│                                                 │
│ [textarea]                                      │
│                                                 │
└─────────────────────────────────────────────────┘
              [ Далее → ]
```

## A5 — Вопросы по одному

Вопросы показываются по одному, не списком.
Все вопросы можно пропустить — поле не обязательно.

```
  ●──────○──────○──────○──────○
  1      2      3      4      5

  ┌───────────────────────────────────────────────┐
  │ Для кого этот результат?                      │
  │                                               │
  │ [textarea]                                    │
  │                                               │
  └───────────────────────────────────────────────┘

  Следующий: «Какой тон и стиль?»   ← превью следующего

  [ ← Назад ]  [ ⚡ Генерировать сейчас ]  [ Далее → ]
```

**Поведение кнопок:**
- **«Далее →»** — переход к следующему вопросу, пустой ответ допустим
- **«⚡ Генерировать сейчас»** — пропустить оставшиеся вопросы, сразу API-вызов
- На 5-м вопросе кнопка **«Далее →»** заменяется на **«Сгенерировать →»**

### Вопросы (статичные)

**Общие для всех (1–4):**

| № | Вопрос |
|---|---|
| 1 | Для кого этот результат? (аудитория) |
| 2 | Какой тон и стиль? (формальный / дружелюбный / технический / и т.д.) |
| 3 | Какой формат вывода? (длина, структура, секции) |
| 4 | Что точно НЕ нужно? (ограничения, табу) |

**Специфичный 5-й вопрос по типу (A2):**

| Тип | Вопрос 5 |
|---|---|
| Контент | Есть ли примеры или референсы, на которые ориентируетесь? |
| Код | На каком языке/стеке? Есть ли архитектурные ограничения? |
| Отчёт | Есть ли шаблон или стандарт оформления (ГОСТ, APA, и т.д.)? |

**→ После 5-го вопроса или «⚡ Генерировать сейчас»:** API-вызов → результат

# ВЕТКА B — Улучшить промпт

## B1 — Вставка промпта

```
┌─────────────────────────────────────────────────┐
│ Вставьте ваш промпт                             │
│                                                 │
│ [textarea — большое]                            │
│                                                 │
└─────────────────────────────────────────────────┘

  [ 🔍 Разобрать по тегам ]   [ ✏️ Преобразовать ]
```

## B2a — Разобрать по XML-тегам

Claude парсит промпт и раскладывает по 6 фиксированным тегам:
`role` · `context` · `task` · `format` · `constraints` · `examples`

*(База тегов — расширяется в будущем)*

```
┌──────────────────────────────────────────────┐
│ <role>                          [↕] [✏️] [🗑] │
│   Ты опытный аналитик данных...              │
│   [editable textarea]                        │
│                                   [📋 копировать тег] │
├──────────────────────────────────────────────┤
│ <context>                       [↕] [✏️] [🗑] │
│   Компания занимается e-commerce...          │
│   [editable textarea]                        │
│                                   [📋 копировать тег] │
├──────────────────────────────────────────────┤
│ <task>                          [↕] [✏️] [🗑] │
│   ...                                        │
│                                   [📋 копировать тег] │
├──────────────────────────────────────────────┤
│ <format>  ...                                │
│ <constraints>  ...                           │
└──────────────────────────────────────────────┘

⚠️  Не найдено: <examples>    ← отсутствующие теги как предупреждение

              [ 📋 Скопировать весь XML ]
```

**Действия для каждого блока:**
- **[↕]** — перетащить / изменить порядок
- **[✏️]** — переименовать тег
- **[🗑]** — удалить блок
- **[📋 копировать тег]** — копирует `<tag>содержимое</tag>` в буфер
- Содержимое редактируется прямо в textarea внутри блока

**Внизу:**
- **«📋 Скопировать весь XML»** — все блоки в порядке как на экране

## B2b — Преобразовать

Выбор из карточек + кастомное поле:

```
┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
│  📋 Добавить     │  │  🗜️ Сжать /      │  │  🌍 Сменить      │
│  XML-структуру   │  │  упростить       │  │  язык            │
└──────────────────┘  └──────────────────┘  └──────────────────┘

┌──────────────────┐  ┌──────────────────┐
│  🎯 Сменить      │  │  ✨ Улучшить     │
│  тон/стиль       │  │  детализацию     │
└──────────────────┘  └──────────────────┘

─────────────────────────────────────────
✏️  Или напишите своё:

┌─────────────────────────────────────────┐
│ Например: «переведи на английский и     │
│ добавь примеры кода»                    │
│ [textarea]                              │
└─────────────────────────────────────────┘
```

*Можно выбрать карточку ИЛИ заполнить поле (или и то и другое)*

→ После выбора / ввода: **[Преобразовать →]** → API-вызов → результат

# Экран результата (общий для A и B)

```
┌─────────────────────────────────────────────────┐
│ // готовый промпт                    N символов │
│                                                 │
│  [текст промпта с typewriter-эффектом]          │
│                                                 │
└─────────────────────────────────────────────────┘

[ 📋 Копировать ]   [ ↺ Уточнить ]   [ 🔄 Начать заново ]
```

**«↺ Уточнить»** — разворачивает поле прямо под промптом:

```
┌─────────────────────────────────────────────────────┐
│ Что изменить?                                       │
│ Например: «сделай короче», «добавь примеры»,        │
│ «более формальный тон», «переведи на английский»    │
│                                                     │
│ [textarea]                                 [ → ]   │
└─────────────────────────────────────────────────────┘
```

Итерировать можно сколько угодно раз подряд.

# Итоговые деревья

```
ВЕТКА A                              ВЕТКА B
──────────────────                   ──────────────────
0. Режим                             0. Режим
   ↓                                    ↓
A1. Новое / Продолжить               B1. Вставка промпта
   ↓                                    ↓
A2. Контент / Код / Отчёт            ┌──────────────┐
   ↓                                 ↓              ↓
A3. Наработки                      B2a. Разбор    B2b. Преобразовать
  (только если A1 = Продолжить)    по XML         (карточки + своё)
   ↓                                 ↓              ↓
A4. Задача (обязательно)             └──── Результат ┘
   ↓
A5. Вопросы ×5 (по одному)
   [Далее] или [⚡ Генерировать сейчас]
   ↓
[API-вызов]
   ↓
Результат
   ↓
[↺ Уточнить — free-form, итеративно]
```
