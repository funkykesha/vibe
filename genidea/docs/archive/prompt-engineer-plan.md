# Prompt Engineer — финальное видение и план реализации

---

## Стек

- **React** (JSX artifact)
- **Tailwind** utility классы
- **Anthropic API** через `fetch` (claude-sonnet-4-20250514)
- **Drag-and-drop** — нативный HTML5 DnD API (десктоп, без библиотек)

---

## Архитектура состояния

```js
{
  // Навигация
  branch: null | 'A' | 'B',
  step: string,              // 'mode' | 'A1' | 'A2' | 'A3' | 'A4' | 'A5' | 'result'
                             //         'B1' | 'B2a' | 'B2b' | 'result'

  // Ветка A
  a: {
    mode: null | 'new' | 'continue',
    type: null | 'content' | 'code' | 'report',
    notes: '',               // наработки (A3)
    task: '',                // задача (A4)
    questionIndex: 0,        // текущий вопрос (0–4)
    answers: ['','','','',''], // ответы на 5 вопросов
  },

  // Ветка B
  b: {
    inputPrompt: '',         // исходный промпт
    xmlBlocks: [],           // [{ id, tag, content, missing }]
    transformCards: [],      // выбранные карточки
    transformCustom: '',     // своё поле
  },

  // Результат (общий)
  result: {
    text: '',
    isEditing: false,        // режим редактирования («Уточнить»)
  },

  // UI
  loading: false,
  loadingMsg: '',
}
```

## Детали экранов

### ModeScreen (0)
- Две большие карточки: «Создать промпт» / «Улучшить промпт»
- Клик → устанавливает `branch`, переход к A1 или B1

### A1 — Новое / Продолжить
- Две карточки
- «Новое» → пропускает A3, идёт к A2 → A4
- «Продолжить» → идёт к A2 → A3 → A4

### A2 — Тип результата
- Три карточки: Контент / Код / Отчёт
- Определяет 5-й вопрос в A5

### A3 — Наработки
- Большой textarea
- Кнопка «Далее →» всегда активна

### A4 — Задача
- Большой textarea
- «Далее →» активируется при ≥20 символах
- Счётчик символов

### A5 — Вопросы
- Прогресс-бар (5 точек, текущая заполнена)
- Один вопрос + textarea
- Превью следующего вопроса (серый, ниже)
- Три кнопки: «← Назад» / «⚡ Генерировать сейчас» / «Далее →»
- На вопросе 5: «Далее →» → «Сгенерировать →»
- «⚡ Генерировать сейчас» — всегда видна начиная с вопроса 1

**Вопросы:**
```js
const QUESTIONS_COMMON = [
  'Для кого этот результат?',
  'Какой тон и стиль?',
  'Какой формат вывода?',
  'Что точно НЕ нужно?',
]
const QUESTION_5 = {
  content: 'Есть ли примеры или референсы, на которые ориентируетесь?',
  code:    'На каком языке/стеке? Есть ли архитектурные ограничения?',
  report:  'Есть ли шаблон или стандарт оформления (ГОСТ, APA и т.д.)?',
}
```

### B1 — Вставка промпта
- Большой textarea (min-height: 200px)
- Две кнопки внизу: «🔍 Разобрать по тегам» / «✏️ Преобразовать»
- Обе активны только при непустом поле

### B2a — XML редактор
- Список блоков, каждый:
  - Заголовок `<tag>` + кнопки [↕ drag handle] [✏️ rename] [🗑 delete]
  - Textarea с контентом (редактируемый)
  - Маленькая кнопка «📋» — копирует `<tag>контент</tag>`
- Drag-and-drop через HTML5 DnD (десктоп)
- Блок «⚠️ Не найдено: …» для отсутствующих тегов из 6 стандартных
- Кнопка внизу «📋 Скопировать весь XML»
- API: Claude получает исходный промпт → возвращает JSON с блоками

**Фиксированные теги:** `role`, `context`, `task`, `format`, `constraints`, `examples`

**Промпт для парсинга:**
```
Разбери следующий промпт на смысловые блоки.
Используй теги из набора: role, context, task, format, constraints, examples.
Верни ТОЛЬКО JSON массив: [{ "tag": "role", "content": "..." }, ...]
Если блок не найден — не включай его. Не добавляй теги которых нет в тексте.
```

### B2b — Преобразовать
- 5 карточек (можно выбрать несколько, тогда суммируются):
  - 📋 Добавить XML-структуру
  - 🗜️ Сжать / упростить
  - 🌍 Сменить язык
  - 🎯 Сменить тон/стиль
  - ✨ Улучшить детализацию
- Textarea «Или напишите своё» — дополняет выбранные карточки
- Кнопка «Преобразовать →» активна если выбрана ≥1 карточка ИЛИ заполнено своё поле
- Логика: `итоговая инструкция = [названия карточек] + [своё поле]` → в один промпт

### ResultScreen
- Промпт в `<pre>` (без typewriter — слишком медленно для длинных промптов, лучше fade-in блоками)
- Счётчик символов
- Кнопки:
  - **«📋 Копировать»** → копирует в буфер, меняется на «✓ Скопировано» на 2 сек
  - **«✏️ Уточнить»** → промпт становится редактируемым textarea
  - **«🔄 Начать заново»** → сброс всего состояния

**Режим «Уточнить»:**
- `<pre>` заменяется на `<textarea>` с тем же текстом
- Пользователь правит вручную
- Кнопки меняются на: «📋 Скопировать» / «✓ Готово» (сохраняет правки и возвращает в режим просмотра)

## API-вызовы

### 1. Генерация промпта (Ветка A)

```
System: Ты элитный prompt engineer. Создай детальный, структурированный промпт
        для мощной AI-модели на основе данных пользователя.
        Структура: [Роль] + [Контекст] + [Задача] + [Формат] + [Ограничения].
        Верни ТОЛЬКО текст промпта, без преамбулы.

User:   Тип задачи: {type}
        Режим: {mode} (новое / продолжение)
        Наработки: {notes или "нет"}
        Задача: {task}
        Аудитория: {answers[0]}
        Тон и стиль: {answers[1]}
        Формат вывода: {answers[2]}
        Ограничения: {answers[3]}
        {question5}: {answers[4]}
```

### 2. Парсинг XML (Ветка B2a)

```
System: Разбери промпт на блоки. Теги: role, context, task, format,
        constraints, examples. Верни ТОЛЬКО JSON: [{"tag":"...","content":"..."}].
        Без markdown, без преамбулы.

User:   {inputPrompt}
```

### 3. Преобразование (Ветка B2b)

```
System: Ты prompt engineer. Преобразуй промпт согласно инструкции.
        Верни ТОЛЬКО изменённый промпт, без преамбулы.

User:   Исходный промпт:
        {inputPrompt}
        
        Инструкция: {cards.join(', ')} {customText}
```

## Порядок реализации

```
1. Скелет: состояние + навигация между экранами (без API)
2. Ветка A: все экраны A1–A5 + логика пропуска A3
3. Ветка B: B1 + B2b (проще чем B2a)
4. API: генерация промпта (A) + преобразование (B2b)
5. ResultScreen: просмотр + копирование + редактирование
6. B2a: XML парсинг + редактор блоков + DnD
7. Полировка: анимации, edge cases, мобильная адаптация (кроме DnD)
```

**Итого API-вызовов в базовом сценарии:** 1  
**Обязательных полей:** A4 (задача) и B1 (промпт)  
**Опциональных полей:** всё остальное
