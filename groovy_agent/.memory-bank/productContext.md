# Product Context

## Products

### groovy_agent

Express-сервер на порту 3000. Проксирует запросы к Yandex Eliza API (OpenAI/Anthropic/internal форматы). Даёт браузеру SSE-стриминг, выполняет Groovy-скрипты.

**Пользователи:** разработчики, пишущие Groovy-скрипты для трансформации JSON с AI-помощью.

**Ключевые сценарии:**
- Написать Groovy-скрипт через чат → получить код → запустить → увидеть stdout
- Выбрать модель из провалидированного списка (probe при старте)
- Поддерживать knowledge base и правила в системном промпте

### genidea

Одностраничный HTML-инструмент (React 18 CDN, без сборки). Генерирует структурированные AI-промпты.

**Пользователи:** контент-менеджеры, аналитики, разработчики — создают промпты без знания prompt engineering.

**Ключевые сценарии (Flow A — новый промпт):**
1. Выбрать тип задачи (content / code / report)
2. Заполнить форму (контекст, задача, аудитория, тон, формат)
3. Нажать «Сгенерировать» → SSE-стриминг через groovy_agent → готовый промпт

**Ключевые сценарии (Flow B — улучшение промпта):**
- B2a: разобрать промпт на XML-блоки (role/context/task/format/constraints)
- B2b: трансформировать промпт по карточке (xml / compress / tone / detailed)
- Уточнить / перевести на EN

**Зависимость:** требует работающего `groovy_agent` на `localhost:3000`.

## Integration

groovy_agent выступает прокси-прослойкой. Оба инструмента используют один `ELIZA_TOKEN`.  
CORS открыт (`origin: '*'`) для dev-режима.

## lib/eliza-client

Извлечённый модуль для повторного использования Eliza-логики в других проектах.  
Публичное API: `createElizaClient({ token, baseUrl })` → `{ getModels, chat, chatOnce, probe }`.  
Пока не опубликован в npm — подключается локально через `package.json` `file:` ссылку.
