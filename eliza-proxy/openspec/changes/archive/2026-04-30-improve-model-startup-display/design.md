## Context

При запуске сервера `eliza-proxy` происходит асинхронный probe моделей для проверки их доступности. Текущая реализация имеет race condition между инициализацией данных моделей и callback-ами probe, что приводит к тому, что модели никогда не обновляют свой статус с ⏳ на ✅/❌. Кроме того, отсутствует инкрементальное обновление интерфейса и перезапись строк в терминале, что требуется по backlog.

Основные проблемы:
1. Race condition между `modelsByProvider` инициализацией и `onModelProbed` callback
2. Отсутствие инкрементального обновления отображения для отдельных моделей
3. Отсутствие перезаписи строк в терминале (только добавление новых)
4. Отсутствие глобального progress bar для всех моделей

## Goals / Non-Goals

**Goals:**
- Устранить race condition между инициализацией данных моделей и probe callbacks
- Реализовать инкрементальное обновление отображения по мере завершения probe каждого model
- Реализовать перезапись строк в терминале вместо добавления новых
- Добавить глобальный progress bar для всех моделей в нижней части вывода
- Обеспечить корректное отображение финальных статусов моделей (✅/❌)

**Non-Goals:**
- Изменение формата вывода в терминале (цвета, символы)
- Изменение логики probe в eliza-client
- Добавление новых API endpoints
- Изменение структуры данных моделей

## Decisions

**Решение: Использовать очередь событий для race condition**
В `onModelProbed` callback: если `modelsByProvider` ещё не заполнен — складывать события в очередь. Когда `modelsByProvider` заполняется, сразу обработать очередь. Это уже частично реализовано, но требует доработки для корректной обработки инкрементальных обновлений.

**Решение: Инкрементальное обновление отображения**
Вместо отображения группы только когда все модели probed, обновлять отображение при каждом завершении probe отдельной модели. Это соответствует требованиям в существующем spec.

**Решение: Перезапись строк в терминале**
Использовать ANSI escape sequences для перемещения курсора и перезаписи строк:
- `\x1b[K` - очистить строку
- `\x1b[1A` - переместить курсор вверх на одну строку
- `\x1b[G` - переместить курсор в начало строки

**Решение: Глобальный progress bar**
Добавить отслеживание общего количества завершенных probe и отображать глобальный progress bar в нижней части вывода.

## Risks / Trade-offs

- [Race condition complexity] → Усложнение логики обработки событий. Митигация: тщательное тестирование различных сценариев запуска.
- [Terminal compatibility] → ANSI escape sequences могут не работать в некоторых терминалах. Митигация: graceful degradation с fallback на обычный вывод.
- [Performance overhead] → Частые обновления терминала могут замедлить startup. Митигация: оптимизация обновлений и минимизация операций вывода.
- [Display flickering] → Перезапись строк может вызывать мерцание. Митигация: точное позиционирование курсора и минимизация перерисовок.

## Migration Plan

Изменения являются обратно совместимыми и не требуют миграции существующих данных или конфигураций. После реализации изменения будут автоматически применены при следующем запуске сервера.
