КОНТЕКСТ ПРОЕКТА:
Ты — Senior Frontend Developer (специалист по PWA и Mobile Web Safari). Твоя задача — написать PWA-приложение для iOS (iPhone) под названием "МотоБардачок" (MotoDocs).
Это безопасное офлайн-хранилище документов на мотоцикл. Приложение должно быть полностью реализовано в ОДНОМ файле index.html (HTML, CSS и Vanilla JS внутри). Никаких сборок, npm, React или сторонних библиотек. Сервера нет. Все работает офлайн.

ТЕХНИЧЕСКИЙ СТЕК И ОГРАНИЧЕНИЯ:

Файл: Строго один index.html. Внутри <style> для CSS и <script> для JS.
База данных: Строго IndexedDB (используй нативную реализацию на Promise). localStorage ЗАПРЕЩЕН для изображений (из-за лимита в 5MB на iOS).
PWA мета-теги: Обязательно добавь теги apple-mobile-web-app-capable, apple-mobile-web-app-status-bar-style (black-translucent). Запрети зум страницы в viewport (user-scalable=no).
Тема: Строгая адаптивная верстка под мобильные устройства с Темной темой (Dark Mode) по умолчанию (черные/темно-серые фоны, белый текст).
АРХИТЕКТУРА ДАННЫХ (IndexedDB):
Создай базу MotoDocsDB. В ней один Store: documents.
Структура записи для ключей (vu, sts, osago, pts):

json
{
  "id": "sts",
  "textData": "Текстовые данные документа",
  "photoFront": Blob, // Сжатое изображение (лицо)
  "photoBack": Blob   // Сжатое изображение (оборот)
}
Также добавь ключ appState с флагом isFirstRun: boolean.

СИСТЕМА РОУТИНГА (Hash Router):
Логика переключений экранов через window.location.hash.

# (или пустой) — Главный экран.
#vu, #sts, #osago, #pts — Экраны просмотра конкретного документа.
#settings — Экран настроек / добавления данных.
#onboarding — Пошаговый мастер первой настройки.
ТРЕБОВАНИЯ К UI И ЭКРАНАМ:

1. Главный экран (#)

Занимает 100% высоты (h-screen). Никаких скроллов.
Три огромные горизонтальные кнопки, каждая по 33% высоты экрана:
ВЕРХ: ВУ (светло-синий/серый акцент)
СРЕДИНА: СТС (розовый/оранжевый акцент)
НИЗ: ОСАГО (зеленый акцент)
Поверх сетки: Иконка настройки ⚙️ (absolute, top-right).
Поверх сетки: Круглая кнопка (FAB) "ПТС" (absolute, bottom-right).
Клик по блоку меняет hash на соответствующий документ.
2. Экран документа (например #sts)

Верхний бар: Огромная кнопка < НАЗАД.
Блок текста: Выводится крупным шрифтом (значение из textData). На iOS 15+ текст отсюда можно легко скопировать.
Блок фото: 2 превью (Лицевая и Оборотная сторона). Клик на превью открывает его на весь экран (Fullscreen PhotoViewer).
3. Fullscreen PhotoViewer (СВЕРХВАЖНО)

Открывается поверх всего, черный фон.
Реализуй нативный Pinch-to-Zoom жестами (события touchstart, touchmove на JS), чтобы пользователь мог двумя пальцами приблизить фото.
Свайп вниз или клик по пустому месту — закрывает просмотрщик.
4. Экран Онбординга (#onboarding) / Настроек (#settings)

Если в DB isFirstRun === true, показываем Онбординг.
Форма для документа содержит:
<textarea class="doc-text"> (на iOS клавиатура автоматически предложит Live Text / Сканирование текста для заполнения этого поля).
<input type="file" accept="image/*"> для фото Лица.
<input type="file" accept="image/*"> для фото Оборота.
ImageProcessor (Логика сжатия): При выборе файла пропусти его через <canvas>, сожми максимальную сторону до 1200px со сбросом качества (например, JPEG 0.8) и сохрани как Blob. Это критично для памяти iPhone!
5. Интеграция с iOS Shortcuts (Команды)
В конце онбординга (и в настройках) выведи блок "Вынести на экран Домой".
У нас есть три захардкоженные ссылки (пока поставь заглушки):
const SHORTCUT_LINKS = { vu: 'https://icloud.com/shortcuts/111', sts: 'https://icloud.com/shortcuts/222', osago: 'https://icloud.com/shortcuts/333' }.

Выведи 3 кнопки в UI: "Установить виджет ВУ", "Установить виджет СТС" и т.д. При клике они открывают соответствующую iCloud-ссылку в новой вкладке.
Shortcut Icon Generator: Добавь мини-утилиту: кнопка "Подготовить картинку для виджета". Пользователь грузит фото мотоцикла, JS рисует его на квадратном <canvas> (500x500px), пишет поверх крупный текст (например "СТС") и рендерит вниз как <img>. Подпиши инструкцию: "Зажмите фото пальцем и выберите 'Сохранить', чтобы использовать его как иконку в приложении Команды".
