# План исправления 5 проблем в МотоБардачок

## Обзор проблем

Пользователь протестировал приложение на iPhone и обнаружил 5 проблем:

1. Не предлагается загрузить документы (фото) при интро
2. Виджет нельзя сделать со своей картинкой
3. Нет кнопки "Добавить на экран" (или запутано)
4. Нет распознавания текста с картинки для внесения текста
5. Даже если добавить команду с картинкой — открывается как URL, а не веб-приложение

---

## Issue 1: Добавить загрузку фото в онбординг

**Проблема:** В онбординге (шаг 2, строки 1178-1197 в `index.html`) есть только `<textarea>` для текста. Нет `<input type="file">` для загрузки фото лицевой и оборотной стороны документов.

**Решение:** Добавить в каждую секцию документа на шаге 2 онбординга:
- `<input type="file" accept="image/*">` для фото лицевой стороны
- `<input type="file" accept="image/*">` для фото оборотной стороны
- Превью загруженных фото
- JS-обработчики для сжатия и временного хранения фото
- Обновить `saveOnboardingData()` чтобы сохранять фото вместе с текстом

**Файлы для изменения:** `index.html`
- HTML: Добавить photo-upload-area блоки в `#onboarding-step-2` для каждого документа (ВУ, СТС, ОСАГО)
- JS: Добавить `onboardingPhotos` объект для временного хранения фото
- JS: Добавить `handleOnboardingPhotoUpload(docId, side, input)` функцию
- JS: Обновить `saveOnboardingData()` — сохранять `photoFront` и `photoBack` из `onboardingPhotos`

## Issue 2: Генератор иконок — выбор документа для надписи

**Проблема:** Генератор иконок использует `currentSettingsDoc` для текста на иконке. В контексте онбординга это может быть не установлено. Пользователь не может выбрать, для какого документа генерировать иконку.

**Решение:** 
- Добавить селектор документа (ВУ/СТС/ОСАГО) перед кнопкой "Выбрать фото" в генераторе иконок
- Генерировать 3 иконки сразу (ВУ, СТС, ОСАГО) из одного загруженного фото
- Показывать все 3 иконки в результате с подписями

**Файлы для изменения:** `index.html`
- HTML: Добавить 3 кнопки-табы для выбора документа в `.icon-generator`
- HTML: Показывать 3 результата (ВУ, СТС, ОСАГО) в сетке
- JS: Обновить `generateIcon()` — генерировать 3 иконки из одного фото
- CSS: Стили для сетки иконок

## Issue 3: Явная кнопка "Добавить на экран Домой"

**Проблема:** После онбординга нет явной инструкции как добавить приложение на главный экран iPhone. Кнопка "Начать использование" сразу завершает онбординг.

**Решение:**
- Добавить шаг 4 в онбординг: "Добавить на экран Домой"
- На этом шаге показать пошаговую инструкцию с иконками:
  1. Нажмите кнопку "Поделиться" (⬆️) внизу Safari
  2. Выберите "На экран Домой"
  3. Нажмите "Добавить"
- Кнопка "Готово" завершает онбординг
- Обновить step-dots с 3 до 4

**Файлы для изменения:** `index.html`
- HTML: Добавить `#onboarding-step-4` с инструкцией
- HTML: Добавить 4-й `step-dot`
- JS: Обновить `updateOnboardingUI()` и `nextOnboardingStep()` для 4 шагов
- CSS: Стили для инструкции (иконки шагов)

## Issue 4: Подсказка про Live Text / сканирование текста

**Проблема:** Пользователь не знает, что iOS 15+ поддерживает Live Text — сканирование текста камерой прямо в textarea.

**Решение:**
- Добавить подсказку-баннер над каждым textarea: "💡 Нажмите на поле и используйте кнопку 📷 на клавиатуре для сканирования текста с документа"
- Добавить атрибут `inputmode="text"` к textarea для лучшей совместимости
- Стилизовать подсказку как информационный блок

**Файлы для изменения:** `index.html`
- HTML: Добавить `.live-text-hint` блок перед каждым textarea в настройках и онбординге
- CSS: Стили для подсказки (иконка, фон, текст)

## Issue 5: PWA манифест и поведение Shortcuts

**Проблема:** Когда iOS Shortcut открывает URL приложения, оно открывается в Safari как обычная веб-страница, а не как standalone PWA.

**Корневая причина:** iOS Shortcuts не могут запускать PWA напрямую. Они могут только открывать URL в Safari. Чтобы приложение работало как standalone PWA, его нужно добавить на главный экран через Safari.

**Решение:**
1. Добавить inline Web App Manifest через `<link rel="manifest">` с data URI:
   ```json
   {
     "name": "МотоБардачок",
     "short_name": "МотоДок",
     "start_url": "./index.html",
     "display": "standalone",
     "background_color": "#0a0a0a",
     "theme_color": "#0a0a0a"
   }
   ```
2. Добавить `apple-touch-icon` (можно сгенерировать через canvas при загрузке или использовать data URI с простой иконкой)
3. Обновить текст в UI: объяснить что Shortcuts — это виджеты для быстрого доступа к конкретному документу, а само приложение нужно добавить через Safari
4. В секции Shortcuts добавить пояснение: "Команды откроют документ в Safari. Для полноэкранного режима добавьте приложение на главный экран через кнопку Поделиться в Safari"

**Файлы для изменения:** `index.html`
- HTML `<head>`: Добавить `<link rel="manifest">` с inline JSON (data URI)
- HTML `<head>`: Добавить `<link rel="apple-touch-icon">` с data URI иконки
- HTML: Обновить текст в shortcuts-section — добавить пояснение
- JS: Добавить генерацию apple-touch-icon через canvas при первой загрузке
